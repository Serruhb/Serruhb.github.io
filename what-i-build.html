Flagship Systems

Two examples that show how I think: signal → context → action.

AI

SYNaTRA — AI Triage Agent

An internal triage assistant that retrieves and summarizes customer history across Salesforce, Jira, and Snowflake, then returns a structured, actionable triage summary for live CS workflows.

  • Core: query building (SOQL/JQL/SQL) + summarization + confidence scoring
  • Outputs: prior fixes, likely matches, next best actions, source references
  • Safety: internal-only guardrails, PII masking, no persistence by default
View case study
OT

OTA Performance Alert & Monitoring

An automated monitoring framework that converts weekly OTA performance metrics into prioritized, owner-aware alerts— catching visibility failures, conversion leakage, and degradation trends early.

  • Signal tiers: visibility → conversion → trend (monitor)
  • Noise control: eligibility thresholds, streaks, baselines to reduce fatigue
  • Ops-ready: alert-to-ticket routing + watchlists + feedback loop for future prediction

How I Work

Low-drama engineering: clear owners, reliable systems, and feedback loops.

01

Workflow first

Define owners, inputs/outputs, and “what good looks like.” Then build the simplest system that works.

02

Reliability built-in

Guardrails, logging, failure modes, and noise control (eligibility rules, thresholds, baselines).

03

Ship & iterate

Deploy small, measure usage, and refine where humans actually get stuck—not where we guess they might.

04

Make it adoptable

Docs + dashboards + handoffs so the system survives staffing changes and stays trusted.
